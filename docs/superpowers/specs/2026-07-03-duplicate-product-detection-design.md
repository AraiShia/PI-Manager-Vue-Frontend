# 订单内重复产品检测设计

## 概述

防止客户因疏忽在同一个订单内对同一产品重复下单。系统在售前预览阶段检测重复并提示用户，导入后在订单详情面板高亮重复行，由用户自行决定保留、跳过或手动删除。

## 设计目标

- **提前发现**：在预览阶段（单条新增、Excel 导入、补充商品）即提示重复。
- **不强制合并**：仅提示和高亮，不自动合并数量或单价，由用户决定。
- **导入后可追溯**：订单详情面板用视觉标记重复行，便于后续处理。
- **最小侵入**：主要改动集中在前端预览表和订单详情面板，后端逻辑不变。

## 检测范围

| 场景 | 触发时机 | 已存在数据来源 |
|---|---|---|
| 单条新增 Dialog | `SingleOrderDialog` 保存后进入预览表 | 当前预览表中的行 |
| Excel 导入预览 | Excel 加载到预览表后 | 当前预览表中的行 |
| 补充商品模式 | Excel/单条数据进入预览表后 | 当前预览表行 + 当前订单已有 items + 客户-产品表已有 Model |
| 订单详情面板 | 每次 `show_order_detail` 渲染时 | 当前订单所有 items |

## 重复判定规则

### 判定键

| 数据类型 | 判定键 | 说明 |
|---|---|---|
| 单条新增行（dict） | `product_id` | `SingleOrderDialog` 返回已选产品对象，若未搜索则为 `None`，不参与判定 |
| Excel 预览行（list） | Model/客户产品编号 列原始值 | 使用 `_find_model_column()` 定位；空值视为无法判定，不参与 |
| 订单详情 item | `product_id` | 若后端未返回，回退到 `customer_code + '|' + oe_number` |
| 当前订单已有 item | `customer_model` / `customer_code` | 补充模式下作为已存在参考 |

### 重复定义

- 同一判定键在待检测集合中出现 **2 次及以上**。
- 判定键为空或 `None` 的行不参与统计，避免误报。

## 预览表交互

### 高亮与状态

检测到重复后：

1. 重复行整行背景色改为浅黄色 `#fef3c7`。
2. 预览状态标签追加提示：`共 X 行数据，其中 Y 行重复产品`。
3. 单条新增时若发现重复，立即弹窗提示。

### 导入确认弹窗

点击“导入”/“确认补充”时，如果存在重复行，弹出 `QMessageBox`：

- 标题：`检测到重复产品`
- 内容：列出重复的产品编号 / OE 号 / Model（最多显示前 10 条）。
- 按钮：
  - **继续导入**：保留所有重复行并导入。
  - **跳过重复行**：仅保留每个重复键的第一次出现，其余不导入。
  - **取消**：返回预览表，让用户手动修改。

### 补充商品模式调整

- 不再自动跳过已存在于当前订单或客户-产品表的行。
- 把这些行标记为重复，并在确认弹窗中一并提示。
- 用户选择“跳过重复行”时，同时跳过预览内重复和与订单/客户-产品表重复的行。

## 订单详情高亮

在 `OrderDetailPanel.show_order_detail()` 渲染完数据行后：

1. 统计所有 item 的 `product_id`（或回退键）出现次数。
2. 出现次数 > 1 的行：
   - 整行背景色设为 `#fef3c7`（浅黄）。
   - 第一列单元格设置 tooltip：`该产品在订单中重复出现`。
3. 优先级：
   - 若某行已因“采购数量不匹配”标红（`#FFF0F0`），红色优先于重复黄色。
   - 合计行不受影响。

## 数据结构与算法

### 重复检测数据结构

```python
{
    "key": "判定键字符串",
    "indices": [0, 3],          # 在预览表中出现的行索引
    "display": "C001 / OE123"   # 用于弹窗展示的产品标识
}
```

### 检测流程

```
输入：rows（预览行列表）、existing_items（可选，当前订单已有 items）
输出：duplicate_groups（重复分组）

1. 初始化 key_map = {}
2. 遍历 rows：
   a. 根据行类型提取 key 和 display
   b. 若 key 为空，跳过
   c. key_map[key].indices.append(index)
   d. key_map[key].display = display
3. 遍历 existing_items（仅补充模式）：
   a. 提取 key
   b. key_map[key] 标记为与外部重复
4. 返回 indices 长度 >= 2 或标记为外部重复的组
```

## 错误处理

- 重复检测逻辑包裹 `try/except`，失败时降级为不提示、不高亮，保证导入流程不中断。
- 空 Model / 空 `product_id` 不参与判定，避免误报。
- 弹窗选择“取消”时，导入流程终止，预览表状态保持不变。

## 测试计划

1. **单条新增重复**：连续两次选择同一产品并保存 → 第二次弹窗提示，预览表高亮。
2. **Excel 重复**：Excel 中同一 Model 出现两次 → 预览表两行均高亮，导入确认弹窗列出重复，选择“跳过重复行”后仅保留一行。
3. **补充模式重复**：补充一个当前订单已存在的产品 → 预览高亮，提示“与订单已有产品重复”。
4. **订单详情高亮**：导入含重复行的订单 → 进入详情面板，重复行浅黄背景并带 tooltip。
5. **编译检查**：`py_compile` 检查改动文件无语法错误。
