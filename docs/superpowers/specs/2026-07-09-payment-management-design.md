# 收款管理页面设计

## 背景

PI Manager 项目当前仅有 `PaymentDialog.vue`（添加付款），缺少独立的收款管理界面。水单图片上传和付款记录查看功能不完整。

## 目标

- 新建独立收款管理页面（路由 `/payments`）
- 管理客户付款（应收），对应 `ArCustomerPayment` 数据
- 支持付款记录列表、添加、编辑、删除、水单上传

## 功能范围

### 已确认
1. 收款列表 — 表格展示所有收款记录，支持搜索/筛选
2. 查看详情 — 右侧滑出面板，显示水单信息+图片预览
3. 添加付款 — Dialog，支持选择PI/日期/金额/付款方式/水单上传
4. 编辑付款 — 在详情面板中编辑
5. 删除付款 — 在详情面板中删除
6. 水单上传 — 支持 jpg/png/pdf，可预览和替换

### 不在本次范围
- 供应商付款管理（应付）
- 收款分摊逻辑
- 工厂定金/尾款同步

## API 调用

| 操作 | API | 说明 |
|------|-----|------|
| 获取收款列表 | `GET /api/payments/receivables` | 支持分页、筛选 |
| 按PI获取收款 | `GET /api/payments/receivables/by-pi/{pi_id}` | 某PI的所有收款 |
| 获取单条收款 | `GET /api/payments/receivables/{id}` | 包含水单图片 |
| 创建收款 | `POST /api/payments/receivables` | 包含水单base64 |
| 编辑收款 | `PUT /api/payments/receivables/{id}` | 可更新水单 |
| 删除收款 | `DELETE /api/payments/receivables/{id}` | 需后端实现 |

## 数据模型

### ArCustomerPayment（收款记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| receipt_no | str | 收据编号（自动生成） |
| pi_id | int | 关联PI |
| customer_id | int | 关联客户 |
| amount | decimal | 应收金额 |
| actual_amount | decimal | 实收金额 |
| handling_fee | decimal | 手续费 |
| payment_date | datetime | 付款日期 |
| payment_method | str | 付款方式 |
| water_image | str | 水单图片(base64) |
| remark | str | 备注 |

## 状态

- 待确认：未关联PI或无水单
- 已确认：有实收金额
- 已结清：实收>=应收

## 验收标准

1. 可通过订单列表页"收款管理"按钮进入页面
2. 收款列表正确显示所有收款记录
3. 可添加新收款（含水单图片）
4. 可编辑已有收款记录
5. 可删除收款记录
6. 水单图片可上传、预览、删除、替换
7. 详情面板滑出显示正确
8. 搜索和筛选功能正常

## 优先级

- P0: 收款列表 + 添加收款 + 水单上传
- P1: 查看详情 + 编辑 + 删除
- P2: 搜索/筛选优化
