# 订单产品编辑 Dialog 设计文档

## 1. 背景与目标

在 PyQt + QWebEngineView + Vue 迁移项目中，订单详情页需要支持对单个 PI item（订单产品）的编辑。本设计覆盖编辑 Dialog 的排版、交互、数据流和实现边界。

## 2. 范围

- 仅覆盖**单个产品 item 的编辑 Dialog**
- 不包含采购/入库独立 Dialog 的内部设计（仅定义打开方式）
- 不包含 Excel 批量导入图片功能（后续独立任务）

## 3. 用户决策摘要

| 问题 | 决策 |
|------|------|
| 编辑对象 | 单个产品 item |
| Dialog 形式 | 居中弹窗，宽度 90% × 高度 90% |
| 业务区块 | 基础信息、销售细节、采购信息、收货入库信息、采购资料存档 |
| 默认展开状态 | 全部展开 |
| 计算字段 | 只读，自动重算 |
| 必填规则 | 按 Excel 截图中 * 号标记 |
| 存储模式 | 即时存储（字段失焦自动保存） |
| 底部按钮 | 仅"关闭" |
| 打开方式 | 双击行 或 右键菜单"编辑产品信息" |
| 右键菜单项 | 编辑产品信息、删除、复制、采购、入库 |
| 图片编辑 | 本地上传 |
| 关闭后行为 | 关闭 Dialog 并刷新订单详情 |

## 4. 设计详情

### 4.1 打开方式

在 `OrderDetailPanel.vue` 的订单产品表格中：

- **双击行**：打开产品编辑 Dialog
- **右键行**：弹出上下文菜单
  - 编辑产品信息
  - 删除
  - 复制
  - 采购 → 打开采购编辑 Dialog
  - 入库 → 打开入库编辑 Dialog

### 4.2 Dialog 规格

```
组件：ProductEditDialog.vue
宽度：90vw
高度：90vh
顶部标题：编辑产品 - {customer_model}
底部按钮：关闭
关闭后：emit('closed') 或 emit('saved')
```

### 4.4 即时存储交互

- 每个可编辑字段使用 `@blur` 触发保存
- 保存调用：`PATCH /api/pi/{pi_id}/items/{item_id}`
- 保存中：字段右侧显示加载图标
- 保存成功：字段右侧显示绿色 ✓（1.5 秒后消失）
- 保存失败：字段边框变红，下方显示错误提示，不自动消失
- 计算字段在相关字段保存成功后自动重新计算

### 4.5 图片上传

- **PyQt 壳环境**：通过 QWebChannel 调用 `window.pyqt.openFileDialog({ accept: 'image/*' })`，返回本地文件路径后上传
- **浏览器环境**：使用 `<input type="file" accept="image/*">` 直接上传
- 上传成功后更新 `image_url` 字段并即时保存

### 4.6 右键菜单

在订单详情表格行上右键弹出：

```
编辑产品信息
删除
复制
采购 → 打开采购编辑 Dialog
入库 → 打开入库编辑 Dialog
```

## 5. 数据流

```
OrderDetailPanel
  ├── 双击行 / 右键菜单 → 打开 ProductEditDialog
  │       └── 字段 blur → PATCH /api/pi/{pi_id}/items/{item_id}
  │       └── 关闭 → emit('closed') → OrderDetailPanel 重新加载 PI detail
  └── 右键菜单"采购"/"入库" → 打开对应独立 Dialog
```

## 6. API 需求

- `PATCH /api/pi/{pi_id}/items/{item_id}` — 更新单个 item 字段
- `POST /api/upload/image` — 图片上传（如后端尚无）
- `GET /api/suppliers/search?keyword=` — 供应商搜索（如后端尚无）

## 7. 错误处理

- 后端返回非 200：字段级显示错误
- 网络失败：字段边框变红，提示"保存失败，请检查网络"
- 必填校验失败：失焦时触发，不提交保存

## 8. 依赖与限制

- 使用 Element Plus 组件：`el-dialog`、`el-form`、`el-input`、`el-input-number`、`el-select`、`el-upload`、`el-message`
- 需要已有的 QWebChannel 桥接支持本地文件选择
- 计算字段的公式必须与后端保持一致
